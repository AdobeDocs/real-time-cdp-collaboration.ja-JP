# 重複数と重複率の計算

Adobe Real-Time CDP Collaborationでは、オーディエンスの重複を理解することは、マーケティング戦略を最適化し、広告主とパブリッシャーの間の効果的な共同作業を確保するために重要です。 このドキュメントでは、サンプルデータを使用して様々な製品領域の重複数と割合を計算する方法について説明します。

## サンプルデータ - Id 数

### 計算目的の前提

この例では、次の条件を想定してください。

* 広告主は、3 つのオーディエンス（A1、A2、A3）を有する。
* パブリッシャーには 3 つのオーディエンス（P1、P2、P3）があります。
* すべての一致するキー ID は、すべてのオーディエンスで相互に排他的です。

>[!NOTE]
>
>実際には、各オーディエンスの一致するキー ID 間でいくつかの重複が発生しますが、簡単にするために、この例ではオーディエンスがこのケースでは排他的であると想定しています。

### 広告主オーディエンス

| 広告主オーディエンス | A1 | A2 | A3 | すべて |
|----------------------|------|------|------|------|
| ハッシュ化されたメール ID | 300K | 450K | 250K | 1M |
| Liveramp ID ID | 500K | 200K | 700K | 140 万 |
| 合計 ID 数 | 800K | 650K | 950K | 240 |

### 公開者オーディエンス

| 公開者オーディエンス | P1 | P2 | P3 | すべて |
|---------------------|------|------|------|------|
| ハッシュ化されたメール ID | 150K | 600K | 550K | 130 万 |
| Liveramp Id ID | 400K | 350K | 100,000 | 850K |
| 合計 ID 数 | 550K | 950K | 800K | 230 万 |

## 重複数と重複率の計算

### サンプルデータ – 重複数

#### 広告主ごと対パブリッシャーごと

|                     | A1 - P1 | A2 - P2 | A3 - P3 |
|---------------------|---------|---------|---------|
| ハッシュ化されたメールによる重複 | 100,000 | 300K | 150K |
| Liveramp Id による重複 | 200K | 150K | 50K |
| 重複の合計 | 300K | 450K | 200K |

#### 広告主それぞれ対パブリッシャーすべて

|                     | A1 - P ALL | A2 - P ALL | A3 - P ALL |
|---------------------|------------|------------|------------|
| ハッシュ化されたメールによる重複 | 250K | 400K | 200K |
| Liveramp Id による重複 | 350K | 150K | 230K |
| 重複の合計 | 600K | 550K | 430K |

#### 広告主 ALL とパブリッシャー EACH

|                     | A ALL - P1 | A ALL - P2 | A ALL - P3 |
|---------------------|------------|------------|------------|
| ハッシュ化されたメールによる重複 | 120K | 530K | 200K |
| Liveramp Id による重複 | 350K | 330K | 50K |
| 重複の合計 | 470K | 860K | 250K |

#### 広告主 ALL とパブリッシャー ALL

|                     | A ALL - P ALL |
|---------------------|---------------|

## Discover モジュール

Adobe Real-Time CDP Collaborationの **[!UICONTROL 検出]** モジュールは、オーディエンスデータに関する貴重なインサイトを提供します。 オーディエンスの重複を理解することで、パブリッシャーと広告主の間の潜在的な共同作業の機会を特定できます。 **[!UICONTROL 検出]** モジュール内の **[!UICONTROL オーディエンスインサイト]** セクションは、異なるオーディエンス間の重複数と割合を分析するのに役立ちます。

![ コラボレーションワークフローの検出モジュール ](/help/assets/reference/overlap-calculations/discover-module-overlap-calculations.png)

様々な重複シナリオについて、サンプルの計算と式を次に示します。

### すべての広告主オーディエンスとすべてのパブリッシャーオーディエンス

| 広告主オーディエンス | 公開者オーディエンス | Id 数（A） | 重複する Id （B） | 重複率 | キーの分類に一致 | キー分類 % に一致 |
|----------------------|---------------------|--------------------|----------------------------|-----------------|---------------------|-----------------------|
| すべて | すべて | すべての広告主オーディエンスの合計 ID 数 <br> ID 数= 1M + 1.4M = 2.4M | すべての一致キーのすべての広告主オーディエンスとすべての公開者オーディエンスの重複合計 <br> ID の重複= 1.58M | すべての広告主オーディエンスの合計 ID 数に対する、重複す <br>ID の割合重複 % = （B/A）* 100 = （1.58M/2.4M）* 100 = 65.83% <br> 重複率= 65.83% | 一致キーごとの重複 ID <br> ハッシュ化されたメールによる重複= 850,00<br>Liveramp ID による重複= 730,000 | 重複する ID の合計に対する一致キーの重複の割合 <br> ハッシュ化されたメールの一致キー % = （850K / 1.58M） * 100 = 53.8% <br> （730K / 1.58M） * 100 = 46.2% |

### すべての広告主オーディエンスと 1 つのパブリッシャーオーディエンス

| 広告主オーディエンス | 公開者オーディエンス | Id 数（A） | 重複する Id （B） | 重複率 | キーの分類に一致 | キー分類 % に一致 |
|----------------------|---------------------|--------------------|----------------------------|-----------------|---------------------|-----------------------|
| すべて | 1 P2 | すべての広告主オーディエンスの合計 ID 数 <br> ID 数= 1M + 1.4M = 2.4M | すべてのマッチキーに関して、すべての広告主オーディエンスと選択したパブリッシャーオーディエンス P2 の重複の合計 <br> ID の重複= 860,000 | すべての広告主オーディエンスの合計 ID 数に対する、重複す <br>ID の割合重複 % = （B/A） * 100 = （860K/2.4M） * 100 = 35.83% <br> 重複率= 35.83% | 一致キーごとの重複 ID <br> ハッシュ化されたメールによる重複= 530,00<br>Liveramp ID による重複= 330,000 | 重複する ID の合計に対する一致キーの重複の割合 <br> ハッシュ化されたメールの一致キー % = （530K / 860K） * 100 = 61.62% <br> （Liveramp ID = （330K / 860K） * 100 = 38.38% |

### 1 人の広告主オーディエンスとすべてのパブリッシャーオーディエンス

| 広告主オーディエンス | 公開者オーディエンス | Id 数（A） | 重複する Id （B） | 重複率 | キーの分類に一致 | キー分類 % に一致 |
|----------------------|---------------------|--------------------|----------------------------|-----------------|---------------------|-----------------------|
| 1 A1 | すべて | 広告主選択オーディエンス A1 の合計 ID 数 <br> ID 数= 300K + 500K = 800K | すべての一致キーの広告主オーディエンス A1 とすべての公開者オーディエンスの重複合計 <br> ID の重複= 600K | 広告主が選択したオーディエンスの ID 数（A1）に対する ID の重複率 <br> 重複率 % = （B/A）* 100 = （600K/800K）* 100 = 75% <br> 重複率= 75% | 一致キーごとの重複 ID <br> ハッシュ化されたメールによる重複= 250,00<br>Liveramp ID による重複= 350,000 | 重複する ID の合計に対する一致キーの重複の割合 <br> ハッシュ化されたメールの一致キー % = （250K/600K） * 100 = 41.67% <br> （Liveramp ID = （350K/600K） * 100 = 58.33% |

### 1 人の広告主オーディエンスと 1 人のパブリッシャーオーディエンス

| 広告主オーディエンス | 公開者オーディエンス | Id 数（A） | 重複する Id （B） | 重複率 | キーの分類に一致 | キー分類 % に一致 |
|----------------------|---------------------|--------------------|----------------------------|-----------------|---------------------|-----------------------|
| 1 A2 | 1 P2 | 広告主選択オーディエンス A2 の合計 ID 数 <br> ID 数= 450K + 200K = 650K | すべてのマッチキーに関して、選択した広告主オーディエンス A2 と選択した公開者オーディエンス P2 の間 <br> 重複の合計 ID が重複= 450K | 選択したオーディエンスの ID 数（A2）に対する ID の重複率 <br> 重複率 % = （B/A） * 100 = （450K/650K） * 100 = 69.23% <br> 重複率= 69.23% | 一致キーごとの重複 ID <br> ハッシュ化されたメールによる重複= 300,00<br>Liveramp ID による重複= 150,000 | 重複する ID の合計に対する一致キーの重複の割合 <br> ハッシュ化されたメールの一致キー % = （300K / 450K） * 100 = 66.67% <br> （Liveramp ID = （150K / 450K） * 100 = 33.33% |
